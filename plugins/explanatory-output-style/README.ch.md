# 解释性输出风格插件

本插件将已弃用的 Explanatory 输出风格重新实现为 SessionStart 钩子。

**警告：** 除非你接受额外指令和输出带来的 Token 消耗，否则请不要安装此插件。

## 功能

启用后，插件会在每次会话开始时自动添加指令，鼓励 Claude：

1. 提供关于实现选择的教育性见解
2. 解释代码库模式和决策
3. 在完成任务的同时提供学习机会

## 工作原理

插件通过 SessionStart 钩子向每个会话注入额外上下文，指示 Claude 在编写代码前后提供简短的教育性解释，格式如下：

    `★ Insight ─────────────────────────────────────`
    [2-3 个关键知识点]
    `─────────────────────────────────────────────────`

## 使用方法

安装后，插件会在每次会话开始时自动激活，无需额外配置。

Insight 侧重于：
- 针对你代码库的具体实现选择
- 代码中的模式和惯例
- 权衡和设计决策
- 代码库特有的细节，而非通用编程概念

## 从输出风格迁移

本插件替代了已弃用的 "Explanatory" 输出风格设置。如果你之前使用：

```json
{
  "outputStyle": "Explanatory"
}
```

现在可以通过安装此插件获得相同的行为。
