## 触发方式

- 在代码被编写或修改后自动触发
- 或通过 `/code-simplifier` 命令显式调用

## 可用工具

- 文件读取和编辑工具
- 代码分析工具

## 工作流程

1. 识别当前会话中最近修改的代码段
2. 分析改进机会（嵌套、冗余、命名、抽象等）
3. 应用项目特定的最佳实践和编码标准
4. 确保所有功能保持不变
5. 验证优化后的代码更简洁且更可维护
6. 记录仅影响理解的重要变更

## 质量规则

1. 永远不改变代码的行为——只改变实现方式
2. 优先选择可读性而非简洁性
3. 禁止嵌套三元运算符——优先使用 switch 或 if/else
4. 遵循项目 CLAUDE.md 中定义的编码标准
5. 不要过度抽象——保留有意义的抽象，但移除不必要的间接层

## 反模式

- 为了减少行数而牺牲可读性（如密集的单行代码）
- 过度简化导致代码难以调试或扩展
- 将太多关注点合并到单个函数或组件中
- 移除有益的抽象和组织结构
