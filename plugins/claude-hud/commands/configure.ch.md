# claude-hud:configure

Claude HUD 显示选项引导式配置。

## 触发条件

- `/claude-hud:configure`
- 用户说"配置 HUD"、"修改状态栏设置"、"自定义 HUD"

## 可用工具

Read, Write, Bash

## 工作流程

### 判断流程

检查 `~/.claude/plugins/claude-hud/config.json` 是否存在：
- **流程 A（新用户）**：布局 → 预设 → 语言 → 关闭项 → 开启项 → 自定义行
- **流程 B（更新配置）**：关闭项 → 开启项 → Git 样式 → 布局/重置 → 语言 → 自定义行

### 预设

三种预设可选：
| 预设 | 启用内容 |
|------|---------|
| **Full** | 全部：工具、Agent、Todo、Git、用量、耗时 |
| **Essential** | 活动行 + Git 状态，信息精简 |
| **Minimal** | 仅核心：模型名称 + 上下文进度条 |

模型名称和上下文进度条始终显示，不可关闭。

### 布局选项

- **Expanded**：多行模式，每组元素独立一行
- **Compact**：单行模式
- **Compact + 分隔符**：单行模式，元素间加分隔符

### 语言

- English (`"en"`)
- 中文 (`"zh-Hans"`)

### Git 样式

- 仅分支名
- 分支 + 脏状态标记 (`*`)
- 完整详情（分支、脏状态、远程超前/落后 `↑N ↓N`）
- 文件统计（额外显示 `!M +A ✘D ?U` 计数）

### 自定义行

可选自由文本行，最长 80 字符。附加在所有其他元素之后。

### 高级字段保护

以下字段不会被引导流程修改，始终保留：`colors.*`、`pathLevels`、时间格式设置、用量/环境/上下文阈值、进度条字符、合并组、元素排序。

### 保存

将修改合并到 `~/.claude/plugins/claude-hud/config.json`。自动将旧格式（`"layout": "default"` 或 `"layout": "separators"`）迁移到新格式。

## 质量规则

1. 确认写入前必须展示预览摘要（布局、语言、Git 变更 + HUD 可视化模拟）
2. 用户取消（Esc）或无任何修改时禁止写入
3. 始终与现有配置合并——禁止盲覆盖
4. 保留引导流程不涉及的所有高级/手动字段

## 反模式

- 不合并直接覆盖现有 config.json
- 在引导流程中编辑高级字段（颜色、阈值、时间格式）
- 未展示预览就确认写入
- 用默认值覆盖用户之前的显式选择
